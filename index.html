1. Gestión de Memoria: El Enlace de Direcciones

La traducción crítica entre variables lógicas de software y celdas físicas de hardware.

Tiempos de Asignación (Binding Time)

  • De Compilación: Se genera código absoluto. Las direcciones físicas finales quedan grabadas de forma estática en el ejecutable.
  • De Liga / Carga: El compilador deja referencias simbólicas y el cargador del S.O. asigna las direcciones físicas en el momento que carga el programa en RAM.
  • De Ejecución: Las direcciones cambian dinámicamente sobre la marcha si el proceso es movido por el kernel. Requiere soporte de hardware (la MMU).

Direcciones Lógicas vs Físicas

Dirección Lógica (Virtual): Generada por la CPU para cada proceso de manera abstracta. El programa asume que cuenta con su propia memoria empezando desde la posición 0.

Dirección Física: La ubicación real y cableada dentro de los chips de memoria RAM.

2. Algoritmos de Ajuste de Memoria

Estrategias de colocación del sistema operativo y su impacto directo en el desperdicio de RAM.

El Problema del Espacio Libre

Fragmentación Externa: Hay suficiente memoria total libre para satisfacer una solicitud, pero no está contigua (bloques dispersos).
Fragmentación Interna: Espacio desperdiciado de forma residual dentro de una partición fija asignada a un proceso que requería menos memoria.

Estrategias Comparadas:

  • Primero (First Fit): Rápido; usa el primer hueco que sirva.
  • Mejor (Best Fit): Minimiza el residuo buscando el hueco más pequeño utilizable.
  • Peor (Worst Fit): Asigna el bloque más grande disponible para dejar remanentes grandes.

3. Sistemas de Archivos: Estructuras Lógicas

Cómo abstrae el sistema operativo el almacenamiento lógico desde la raíz del sistema.

Modelos Jerárquicos

Estructura de Árbol:

Estructura jerárquica con un nodo raíz (/). Evita colisiones de nombres gracias al uso de rutas absolutas y relativas, y permite delegar permisos por directorios.

Estructura de Grafos Acíclicos:

Permite ramificar directorios que apuntan de forma compartida al mismo archivo físico en el disco sin duplicar los bytes. Se implementa en Linux/Unix usando enlaces.

i-nodo (Nodo de Índice): Estructura física de datos en disco que guarda los metadatos esenciales del archivo: tamaño, UID (dueño), GID (grupo), permisos y MAC times (timestamps), pero no almacena el nombre del archivo.

4. Métodos de Acceso y Colisiones Hash

Cómo lee el kernel la información física y cómo el indexado por hash acelera el rendimiento.

Mecanismos de Lectura

  • Secuencial: Lectura lineal de registros uno por uno. Costo $O(N)$. Típico para lectura de logs de auditoría o procesamiento por lotes.
  • Indexado: Un archivo índice almacena punteros rápidos para realizar saltos directos de lectura en el archivo principal.
  • Acceso por Hash: Calcula la dirección física del bloque mediante una función matemática, logrando una velocidad constante $O(1)$.

5. Gestión de Procesos: PCB y anatomía en Memoria

Cómo vive y se estructura un binario activo bajo el control directo del kernel.

La Abstracción de un Proceso

Es la representación que hace el S.O. de un programa en ejecución activa, con recursos físicos asignados por el programador.

Estructura en la RAM

  • Segmento Texto: Contiene las instrucciones compiladas en lenguaje de máquina (De solo lectura).
  • Segmento Datos: Guarda variables globales e inicializadas.
  • Heap (Montículo): Memoria asignada dinámicamente en ejecución (malloc / new). Crece hacia arriba.
  • Stack (Pila): Parámetros de funciones, variables locales y direcciones de retorno de llamadas. Crece hacia abajo.
PCB: Bloque de Control de Procesos

6. Estados, Transiciones e Hilos de Procesos

El ciclo de vida de un hilo de ejecución y la asignación del despachador del procesador.

¿Qué es un Hilo (Thread)?

La unidad básica de uso de CPU que comparte recursos del proceso padre (como código y variables globales) pero contiene su propia pila y contador de programa para operar de forma concurrente.

Algoritmos de Planificación

  • Round Robin: Rota el procesador de forma equitativa asignando un intervalo de tiempo llamado Quantum.
  • FCFS (First Come First Served): Atiende por estricto orden de llegada.

7. Gestión de Usuarios y Active Directory

Seguridad perimetral y administración centralizada de identidades en el sistema operativo.

Arquitectura de Directorio

La base de la seguridad empresarial reside en no autorizar accesos de forma local sino centralizada:

Directorio Activo (AD): Estructura jerárquica de base de datos que administra usuarios y políticas mediante consultas LDAP y autenticación segura Kerberos basada en tickets de cifrado.
Políticas de Grupo (GPO): Políticas de configuración centralizadas administradas por el S.O. para aplicar restricciones masivas en estaciones y servidores.

8. Técnicas de Protección del Sistema Operativo

El blindaje del kernel y la asignación de privilegios en bajo nivel.

Protecciones a Nivel de Kernel

DEP / NX (Data Execution Prevention):

Marca zonas de memoria como no ejecutables (ej. la pila). Si un exploit inyecta código de ejecución en el Stack, la CPU cancela el proceso de inmediato.

ASLR (Address Space Layout Randomization):

Aleatoriza las ubicaciones donde se cargan los segmentos del proceso en RAM cada vez que corre. Evita que un atacante prediga direcciones fijas del sistema.

Estrategias de Seguridad de Acceso

Modelos de Privilegios

DAC (Discretionary Access Control): El propietario de un archivo decide de forma autónoma quién tiene permisos. Sencillo pero propenso a errores humanos de permisos flojos.
MAC (Mandatory Access Control): El sistema operativo evalúa directivas globales de seguridad rígidas e inmutables (ej. SELinux). Aunque el dueño quiera, no puede dar permisos que vulneren las reglas globales.
Conclusión del Bloque: La combinación de DEP, ASLR y esquemas MAC de alta rigidez convierte a los kernels modernos en fortalezas robustas, forzando a los cibercriminales a buscar vulnerabilidades de lógica web o ingeniería social.
