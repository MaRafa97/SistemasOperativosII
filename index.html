1. Gestión de Memoria: El Enlace de Direcciones

Relocalización dinámica y aislamiento de hardware mediante protección por registros de límites.

Tiempos de Enlace (Binding Times)

La conversión de variables simbólicas en direcciones físicas reales pasa por tres fases críticas de la compilación y ejecución:

  • Tiempo de Compilación: Si se conoce a priori la dirección de carga física en RAM, el compilador genera código absoluto. Si la dirección de inicio varía, el software arrojará un fallo catastrófico obligando a una recompilación total.
  • Tiempo de Carga (Liga): El compilador produce direcciones relocalizables en código objeto. El cargador (loader) del sistema operativo calcula las direcciones físicas absolutas justo al colocar el software en la RAM.
  • Tiempo de Ejecución: Si el proceso se mueve entre segmentos de RAM durante la ejecución, el enlace se aplaza hasta el uso del operando. Requiere soporte de hardware (MMU).

Espacios de Direccionamiento

Dirección Lógica: Dirección generada virtualmente por la CPU. El mapa lógico abarca un rango exclusivo $[0, \text{Límite}-1]$ desde la perspectiva del proceso.

Dirección Física: Dirección absoluta cableada e indexada en los circuitos y bancos de la RAM.

La Unidad de Gestión de Memoria (MMU): Módulo físico de hardware que implementa la traslación aritmética instantánea mediante el Registro de Relocalización (Base): $Dirección\_Física = Registro\_Base + Dirección\_Lógica$

2. Algoritmos de Ajuste de Memoria

Degradación física de la memoria primaria mediante asignación dinámica y fragmentación.

Análisis Formal de Fragmentación

Fragmentación Externa:

La suma de todos los huecos libres satisface el proceso, pero ningún hueco contiguo individual es suficiente: $$\sum \text{Huecos\_Libres} \ge \text{Tamaño\_Proceso}$$ $$\forall h_i \in H_L, \quad h_i < \text{Tamaño\_Proceso}$$

Fragmentación Interna:

Desperdicio dentro de la partición asignada — el bloque es mayor que el proceso: $$\Delta_{int} = Tamaño\_Bloque - Tamaño\_Proceso$$

Compactación (Solución):

El SO reorganiza físicamente los procesos en RAM consolidando todos los huecos dispersos en un único bloque contiguo. Requiere soporte de relocalización dinámica (MMU).

Comparativa de Algoritmos
First Fit Primera cavidad ≥ proceso. Más rápido.
Best Fit Menor hueco residual. Minimiza frag. interna.
Worst Fit Mayor hueco residual. Maximiza remanente útil.

3. Sistemas de Archivos: Estructuras Lógicas

De la raíz del sistema de directorios a los grafos lógicos acíclicos y la persistencia física.

Modelos Jerárquicos Lógicos

Estructura de Árbol (Tree-Structured):

Sistema jerárquico estándar. Se hereda desde el directorio raíz y se define la ruta absoluta del recurso (ej: /usr/bin/python). Resuelve la colisión de nombres duplicados aislando los espacios de nombres.

Estructura de Grafos Acíclicos (Acyclic-Graph):

Permite que varios subdirectorios tengan enlaces que apunten al mismo archivo físico de forma compartida. Evita la duplicación en disco pero incrementa la complejidad del recorrido y control de punteros (riesgo de referencias circulares si se permiten grafos no acíclicos).

Tipos de Enlaces en Unix / Linux

  • Enlaces Duros (Hard Links): Registros de directorio que apuntan al mismo número de inodo. El archivo original no se elimina hasta que el contador de referencias (link count) llegue a cero.
  • Enlaces Simbólicos (Symlinks): Archivo independiente cuyo contenido es la ruta literal de otro archivo. Si el objetivo es eliminado, el symlink queda como una referencia rota (dangling pointer).
Estructura de Metadatos del Kernel

i-nodo (Nodo de Índice) y Descriptores

Estructura del Inodo (EXT4): Guarda la metadata técnica del archivo: Tamaño del archivo en bytes, ID del propietario (UID), ID del grupo (GID), Permisos POSIX (rwxrwxrwx), marcas de tiempo (Access, Modify, Change - MAC) y los bloques físicos que mapean el contenido. Nota: El inodo físico no conoce el nombre del archivo; este se mapea únicamente en la tabla lógica del directorio.

4. Métodos de Acceso y Colisiones Hash

Análisis computacional del acceso físico e indexación mediante direccionamiento por hash de velocidad constante $O(1)$.

Estrategias de Búsqueda

  • Secuencial: El puntero de lectura avanza byte por byte o registro por registro desde el inicio. Costo computacional $O(N)$ en el peor de los casos.
  • Aleatorio / Directo: La CPU calcula directamente la dirección del bloque de disco utilizando el tamaño de registro fijo $R$: $Offset = n \times R$. Velocidad de acceso constante $O(1)$.
  • Indexado: Utiliza una tabla índice intermedia. Se realiza una búsqueda binaria en el índice ($O(\log N)$) para obtener el puntero de salto directo al archivo físico.

5. Gestión de Procesos: PCB y anatomía en Memoria

Mapeo del espacio de memoria de un proceso y el Bloque de Control de Procesos (PCB) en el kernel.

Estructura del Proceso en la RAM

Cuando el cargador del sistema operativo mapea un programa ejecutable en la RAM, divide el espacio lógico de direcciones del proceso en cuatro segmentos específicos:

  • Texto (Code): Las instrucciones compiladas en lenguaje de máquina. Es de solo lectura para evitar autodesbordamientos o modificaciones maliciosas.
  • Datos (Initialized/BSS): Variables globales e inicializadas en el código del programa.
  • Heap (Montículo): Memoria reservada de manera dinámica en ejecución por el programador (malloc / new). Crece hacia arriba.
  • Stack (Pila): Datos temporales para la ejecución de funciones (variables locales, parámetros y direcciones de retorno de saltos). Crece hacia abajo de forma LIFO.
PCB: Process Control Block (task_struct)

El sistema operativo acumulará punteros a objetos en la memoria protegida del Kernel hasta causar una **Denegación de Servicio (DoS)** interna, obligando al reinicio total del host físico.

6. Estados, Transiciones e Hilos de Procesos

El despachador (Scheduler), las unidades básicas de ejecución y la asincronía de hilos concurrentes.

¿Qué es un Hilo (Thread)?

Es la unidad básica de asignación de tiempo del procesador (CPU). Un hilo opera de forma concurrente compartiendo con sus hilos hermanos del mismo proceso el segmento de código (Texto), datos y descriptores lógicos del sistema (archivos abiertos), pero mantiene de forma exclusiva su propio **Program Counter (PC)**, **Pila de ejecución (Stack)** y **Registros de contexto**.

Planificadores de la CPU

  • Round Robin (RR): Modelo preventivo basado en tiempo compartido. Asigna una rodaja de tiempo fija llamada **Quantum ($q$)**. Al expirar el quantum, el Scheduler interrumpe al proceso activo enviándolo a la cola de Listo.
  • FCFS (First Come First Served): No preventivo. Atiende en orden de llegada directo. Sufre del "Efecto Convoy", donde procesos cortos esperan por uno largo.

7. Gestión de Usuarios y Active Directory

Arquitecturas federadas de identidad, autenticación cifrada por tickets y hardening centralizado.

Servicios de Directorio Corporativos

Para la gestión segura de miles de hosts en redes corporativas, los sistemas de usuarios locales se delegan a un sistema de base de datos jerárquico y distribuido:

Active Directory (AD): Estructura jerárquica de Microsoft que implementa consultas centralizadas mediante protocolo LDAP (Puerto TCP/389) y autentica de forma segura usando el protocolo **Kerberos** (Puerto UDP/88), basado en tickets criptográficos simétricos para evitar la transmisión de contraseñas en texto claro por la red.
GPO (Group Policy Object): Directivas de grupo que aplica el kernel de Windows de manera centralizada. Definen las plantillas de seguridad del sistema de archivos, el registro, las directivas de red y las restricciones de ejecución de procesos.

8. Técnicas de Protección del Sistema Operativo

Mitigación de exploits de memoria en el procesador y modelos formales de acceso.

Técnicas de Protección del Kernel

DEP / NX (Prevención de Ejecución de Datos):

Mecanismo a nivel de hardware/software donde la CPU marca de manera forzada ciertas páginas de memoria (como la pila/Stack y el montículo/Heap) con el bit NX (No-Execute). Si un exploit inyecta código ensamblador (Shellcode) en el Stack mediante desbordamiento e intenta saltar a ejecutarlo, el microprocesador cancela instantáneamente el proceso.

ASLR (Aleatorización del Espacio de Memoria):

Aleatoriza de forma dinámica las posiciones en memoria RAM de los componentes clave de un proceso (Stack, Heap, código ejecutable y librerías dinámicas del sistema de archivos como libc). Evita que un exploit use direcciones estáticas de saltos predecibles en memoria.

Modelos de Control de Acceso

Políticas, Procedimientos y Modelos Formales

DAC (Discretionary Access Control): El propietario de un recurso físico o lógico tiene la facultad discrecional de otorgar o revocar permisos de acceso (ejemplo: permisos estándar `chmod` de Linux). Cómodo, pero propenso a errores humanos de asignación.
MAC (Mandatory Access Control): El sistema operativo restringe de forma inflexible el acceso basándose en etiquetas de seguridad globales de Kernel (ejemplo: SELinux o AppArmor). Aunque el usuario intente dar permisos a un archivo, si viola las directivas MAC corporativas, el acceso es denegado de forma rígida.
Conclusión de Hardening: Un sistema operativo seguro combina defensas activas del procesador (DEP + ASLR) con modelos de control de privilegios rígidos (MAC) para aislar por completo las brechas de exploits en bajo nivel.
